# 1. PostgreSQL 소개
#PostgreSQL/Introduction

> [!INFO]
> PostgreSQL은 **"세계에서 가장 발전된 오픈소스 객체 관계형 데이터베이스"**로 불리는 데이터베이스 관리 시스템(ORDBMS)입니다. 1996년에 처음 출시된 이후 30년이 넘는 기간 동안 활발한 오픈소스 커뮤니티에 의해 개발되어 왔으며, 뛰어난 안정성, 강력한 기능, 높은 확장성을 자랑합니다.

## 1.2. 역사

- **1986년**: 캘리포니아 버클리 대학의 마이클 스톤브레이커 교수가 이끈 **POSTGRES** 프로젝트에서 시작되었습니다. 이는 기존 관계형 데이터베이스의 한계를 극복하고 객체 지향 개념을 도입하려는 시도였습니다.
- **1994년**: POSTGRES에 SQL 언어 인터프리터를 추가한 **Postgres95**가 개발되었습니다.
- **1996년**: 프로젝트가 오픈소스 커뮤니티로 이전되면서, SQL 표준을 준수한다는 의미를 강조하기 위해 **PostgreSQL**로 이름이 변경되었습니다.
- **1997년**: PostgreSQL 6.0 버전이 출시된 이후, 전 세계 개발자들의 기여로 빠르게 발전하며 현재에 이르고 있습니다.

## 1.3. 주요 특징

- **객체 관계형 데이터베이스 (ORDBMS)**: 관계형 데이터베이스의 기능에 더해, 테이블 상속, 사용자 정의 타입 등 객체 지향적인 특징을 지원하여 복잡한 데이터 구조를 효과적으로 다룰 수 있습니다.
- **오픈소스 및 자유로운 라이선스**: BSD와 유사한 유연한 라이선스 정책을 채택하여, 상업적인 용도로 사용하더라도 라이선스 비용 없이 자유롭게 사용, 수정, 배포할 수 있습니다.
- **뛰어난 표준 준수**: 가장 최신 SQL 표준인 `SQL:2023`의 대부분의 기능을 지원할 만큼 SQL 표준을 매우 잘 준수합니다.
- **강력한 확장성**: 사용자가 직접 데이터 타입, 함수, 연산자 등을 정의할 수 있으며, 다양한 확장 기능(Extension)을 통해 데이터베이스의 기능을 무한히 확장할 수 있습니다. (예: 지리 정보 시스템을 위한 `PostGIS`)
- **ACID 트랜잭션 보장**: 데이터베이스의 핵심 원칙인 원자성(Atomicity), 일관성(Consistency), 격리성(Isolation), 지속성(Durability)을 완벽하게 보장하여 데이터의 무결성과 신뢰도를 높입니다.
- **다양한 데이터 타입 지원**: 숫자, 문자열, 날짜와 같은 기본 타입 외에도 배열(Array), `JSON`/`JSONB`, `XML`, `HSTORE`(Key-Value) 등 복합적인 데이터를 처리하기 위한 풍부한 데이터 타입을 내장하고 있습니다.
- **다중 버전 동시성 제어 (MVCC)**: 읽기 작업과 쓰기 작업이 서로를 차단하지 않는 MVCC 방식을 사용하여, 다중 사용자 환경에서도 높은 동시성과 성능을 제공합니다.

## 1.4. 장점

- **높은 안정성과 신뢰성**: 오랜 개발 역사와 검증을 통해 매우 안정적이며, 중요한 비즈니스 데이터를 안전하게 저장하고 관리할 수 있습니다.
- **비용 절감**: 라이선스 비용이 없어 초기 도입 및 유지보수 비용을 크게 절감할 수 있습니다.
- **복잡한 쿼리 처리 능력**: 복잡하고 분석적인 쿼리에서 뛰어난 성능을 발휘하며, 대규모 데이터 웨어하우스(DW) 환경에도 적합합니다.
- **활발한 커뮤니티**: 전 세계의 활발한 커뮤니티로부터 지속적인 기술 지원과 업데이트를 받을 수 있습니다.
- **Oracle과의 높은 호환성**: 상용 데이터베이스인 Oracle과 유사한 기능과 구조를 많이 가지고 있어, Oracle에서 마이그레이션하기 좋은 대안으로 꼽힙니다.
