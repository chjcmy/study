# 00. 트리 기반 모델 (Tree-Based Models)

## 1. 핵심 정의

**트리 기반 모델(Tree-Based Models)**은 데이터에 대한 질문을 연속적으로 던져 데이터를 작은 그룹으로 나누어 나가는 **의사 결정 트리(Decision Tree)** 구조를 기본 블록으로 사용하는 머신러닝 알고리즘 그룹입니다.

-   **동작 방식**: 전체 데이터를 가장 잘 구분하는 질문(특성)을 찾아 두 개 이상의 하위 그룹으로 분할합니다. 이 과정을 각 하위 그룹에 대해 반복하여, 최종적으로 각 그룹이 가능한 한 동일한 클래스(분류) 또는 유사한 값(회귀)을 갖도록 합니다.
-   **시각화**: 전체 구조가 나무를 거꾸로 뒤집어 놓은 것 같은 형태를 띠기 때문에 '트리(Tree)' 모델이라고 불립니다.

## 2. 주요 트리 기반 모델

-   **[[01. 의사 결정 트리]]**: 가장 기본이 되는 모델입니다. 단일 트리를 사용하여 예측을 수행하며, 해석이 쉽다는 장점이 있습니다.

-   **앙상블(Ensemble) 모델**: 여러 개의 의사 결정 트리를 결합하여 단일 트리의 단점(특히 과적합)을 보완하고 예측 성능을 극대화하는 기법입니다.
    -   **[[02. 랜덤 포레스트]] (Random Forest)**: 여러 개의 의사 결정 트리를 독립적으로 학습시킨 후, 그 결과를 종합(투표 또는 평균)하여 최종 예측을 결정합니다. (배깅, Bagging 방식)
    -   **[[03. 그래디언트 부스팅 머신|그래디언트 부스팅 (Gradient Boosting)]]**: 간단한 트리를 순차적으로 학습시키면서, 이전 트리가 놓친 오차를 다음 트리가 보완해 나가는 방식으로 점진적으로 모델 성능을 향상시킵니다. (부스팅, Boosting 방식)
        -   **XGBoost, LightGBM, CatBoost**: 그래디언트 부스팅을 더욱 발전시킨 고성능 알고리즘들입니다.

## 3. 트리 기반 모델의 특징

### 장점

-   **해석의 용이성**: 특히 단일 의사 결정 트리의 경우, 모델이 왜 그런 결정을 내렸는지 이해하기 쉽습니다 (화이트박스 모델).
-   **적은 데이터 전처리**: 데이터의 스케일에 영향을 받지 않아 스케일링이 필요 없는 경우가 많습니다.
-   **다양한 데이터 유형 처리**: 수치형 데이터와 범주형 데이터를 모두 다룰 수 있습니다.

### 단점

-   **과적합 경향**: 단일 트리는 데이터의 노이즈까지 학습하여 [[과적합]]되기 쉽습니다. (앙상블 모델로 보완)
-   **불안정성**: 학습 데이터가 약간만 변해도 트리의 구조가 크게 바뀔 수 있습니다.
