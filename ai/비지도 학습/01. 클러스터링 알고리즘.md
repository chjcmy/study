# 01. 클러스터링 알고리즘 (Clustering Algorithms)

## 1. 클러스터링이란?

**클러스터링(Clustering)**은 **유사성을 기반으로 데이터 포인트를 자동으로 그룹화**하는 **비지도 학습(Unsupervised Learning)** 기법입니다. 레이블이 없는 데이터에서 독립적으로 패턴을 찾아 클러스터(그룹)를 형성합니다.

### 분류(Classification)와의 차이점

-   **분류 (Classification)**: 레이블이 있는 데이터로 학습(지도 학습)하며, 새로운 데이터의 범주를 예측합니다. (예: 스팸 메일 분류)
-   **클러스터링 (Clustering)**: 레이블 없이 유사한 데이터끼리 자동으로 그룹화(비지도 학습)합니다. (예: 고객 세분화)

## 2. 클러스터링 활용 분야

클러스터링은 다양한 분야에서 데이터의 숨겨진 구조를 발견하고 통찰력을 얻는 데 활용됩니다.

-   **고객 세분화**: 구매 패턴, 인구 통계학적 정보 등을 기반으로 고객을 여러 그룹으로 나누어 타겟 마케팅 전략을 수립합니다.
-   **이상 탐지 (Anomaly Detection)**: 정상적인 데이터 그룹에서 벗어나는 데이터 포인트를 찾아 사기 탐지, 장비 오작동 감지 등에 활용합니다.
-   **이미지 분할 (Image Segmentation)**: 이미지 내에서 유사한 픽셀들을 그룹화하여 객체 인식, 의학적 이상 감지, 패턴 인식 등에 사용됩니다.
-   **특징 엔지니어링 (Feature Engineering)**: 클러스터링 결과를 새로운 특징으로 활용하거나, 데이터의 차원을 축소하는 데 기여합니다.
-   **데이터 압축**: 유사한 데이터를 하나의 클러스터로 묶어 데이터 크기를 줄이는 데 사용됩니다. (예: 이미지 압축)
-   **음악/콘텐츠 분류**: 장르 식별, 추천 시스템 등에서 유사한 콘텐츠를 그룹화합니다.

## 3. 주요 클러스터링 알고리즘 종류

### 3.1. 파티션 기반 클러스터링 (Partition-based Clustering)

-   **대표 알고리즘**: K-means (K-평균)
-   **특징**: 데이터를 겹치지 않는 $K$개의 클러스터로 분할하며, 각 클러스터 내의 데이터 분산(거리)을 최소화하는 것을 목표로 합니다.
-   **장점**: 구현이 간단하고 효율적이며, 대규모 데이터셋에 적합합니다.
-   **단점**: 클러스터의 개수 $K$를 미리 지정해야 하며, 불규칙한 형태의 클러스터나 밀도가 다른 클러스터는 잘 감지하지 못합니다. 이상치에 민감합니다.

### 3.2. 밀도 기반 클러스터링 (Density-based Clustering)

-   **대표 알고리즘**: DBSCAN (Density-Based Spatial Clustering of Applications with Noise)
-   **특징**: 데이터 밀도를 기준으로 클러스터를 형성합니다. 밀집된 영역을 클러스터로 간주하고, 밀도가 낮은 영역의 데이터는 잡음(noise)으로 처리합니다.
-   **장점**: 클러스터의 개수를 미리 지정할 필요가 없으며, 불규칙한 형태의 클러스터 탐지가 가능하고, 잡음 데이터 처리에 강합니다.
-   **예시**: 반원형과 같이 복잡한 형태의 클러스터도 효과적으로 분리할 수 있습니다.

### 3.3. 계층적 클러스터링 (Hierarchical Clustering)

데이터를 중첩된 트리 구조(덴드로그램, Dendrogram)로 구성하며, 두 가지 주요 방식이 있습니다.

#### 1) 응집 계층적 클러스터링 (Agglomerative Hierarchical Clustering)

-   **상향식 접근 (Bottom-up)**: 각 데이터 포인트가 개별 클러스터로 시작하여, 가장 유사한 클러스터끼리 점진적으로 병합해 나갑니다.
-   **과정**:
    1.  각 데이터 포인트를 $N$개의 개별 클러스터로 시작합니다.
    2.  모든 클러스터 쌍 간의 거리(유사도)를 계산하는 거리 행렬을 만듭니다.
    3.  가장 가까운 두 클러스터를 하나의 새로운 클러스터로 병합합니다.
    4.  거리 행렬을 업데이트한 후, 원하는 클러스터 개수에 도달하거나 더 이상 병합할 수 없을 때까지 2~3단계를 반복합니다.

#### 2) 분할 계층적 클러스터링 (Divisive Hierarchical Clustering)

-   **하향식 접근 (Top-down)**: 전체 데이터를 하나의 큰 클러스터로 시작하여, 가장 이질적인 클러스터를 점차 더 작은 클러스터로 분할해 나갑니다.
-   **과정**:
    1.  모든 데이터를 포함하는 하나의 클러스터로 시작합니다.
    2.  클러스터를 가장 이질적인 두 개의 하위 클러스터로 분할합니다.
    3.  미리 정해진 중지 기준(예: 클러스터의 최소 크기, 클러스터 개수)에 도달할 때까지 2단계를 반복합니다.

## 4. 포트폴리오 적용 팁

클러스터링은 실제 비즈니스 문제 해결에 매우 유용하게 활용될 수 있습니다.

-   **고객 행동 모델링**: H&M 고객 데이터와 같은 실제 데이터를 사용하여, K-means로 고객을 3~5개 그룹으로 세분화(구매 패턴, 선호 카테고리 기준)할 수 있습니다.
-   **유사도 분석**: 계층적 클러스터링을 통해 고객 간의 유사도 트리를 생성하고, 이를 기반으로 고객 그룹의 특성을 시각화할 수 있습니다.
-   **이상 탐지**: DBSCAN을 활용하여 일반적인 구매 패턴에서 벗어나는 이상 구매 패턴(예: 사기)을 탐지할 수 있습니다.

클러스터링 결과를 시각화하고, 각 그룹의 특성을 해석하여 **타겟 마케팅 전략**이나 **추천 시스템**으로 연결하면 실무형 포트폴리오를 구축하는 데 큰 도움이 됩니다.
