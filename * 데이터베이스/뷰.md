# 데이터베이스 뷰 (View)
#Database/View

## 개요

> [!INFO]
> 뷰(View)는 하나 이상의 테이블에서 유도된 **가상 테이블(Virtual Table)** 입니다. 실제 데이터를 저장하지 않고, 뷰가 참조하는 쿼리 정의만 저장합니다. 복잡한 쿼리를 단순화하고 데이터 접근을 추상화하는 데 사용됩니다.

## 특징

1.  **가상성**: 실제 데이터를 저장하지 않고, 쿼리가 실행될 때 동적으로 결과를 생성합니다.
2.  **동적 업데이트**: 기반이 되는 원본 테이블의 데이터가 변경되면 뷰의 결과도 자동으로 반영됩니다.
3.  **보안성**: 사용자에게 테이블의 특정 행이나 열만 노출시켜 데이터 접근을 제어하는 데 활용할 수 있습니다.
4.  **쿼리 단순화**: 여러 테이블을 조인하는 복잡한 쿼리를 뷰로 만들어두면, 사용자는 단순한 `SELECT` 문으로 해당 결과를 조회할 수 있습니다.

## 장점과 단점

-   **장점**
    1.  **데이터 독립성**: 뷰를 사용하는 애플리케이션은 실제 테이블 구조의 변경에 영향을 덜 받습니다.
    2.  **재사용성**: 복잡한 쿼리를 반복적으로 사용할 필요 없이 뷰를 재사용할 수 있습니다.
    3.  **보안 강화**: 중요한 데이터나 민감한 정보를 숨기고, 허용된 데이터만 뷰를 통해 노출할 수 있습니다.

-   **단점**
    1.  **성능 이슈**: 뷰의 정의가 너무 복잡하면 쿼리 실행 시 성능이 저하될 수 있습니다.
    2.  **제한된 업데이트**: 여러 테이블을 조인하는 등 복잡한 뷰는 `INSERT`, `UPDATE`, `DELETE` 작업이 불가능하거나 제한될 수 있습니다.

## 뷰 생성 및 사용 예시

```sql
-- 급여가 50000 이상인 직원들만 보여주는 뷰 생성
CREATE VIEW high_salary_employees AS
SELECT employee_id, first_name, last_name, salary
FROM employees
WHERE salary > 50000;
```

```sql
-- 생성된 뷰를 일반 테이블처럼 사용
SELECT * FROM high_salary_employees;
```

## 뷰의 종류

1.  **단순 뷰 (Simple View)**: 단일 테이블에 기반한 뷰.
2.  **복합 뷰 (Complex View)**: 여러 테이블을 조인하거나 집계 함수를 사용한 뷰.
3.  **인라인 뷰 (Inline View)**: `FROM` 절에서 사용되는 서브쿼리. 일회성으로 사용되는 임시 뷰입니다.
4.  **Materialized View**: 일반 뷰와 달리 실제 데이터를 저장하는 특수한 뷰. 데이터 조회 성능은 빠르지만, 원본 데이터 변경 시 주기적으로 데이터를 새로고침(`REFRESH`)해야 합니다.

> [!CAUTION] 주의사항
> - 복잡한 뷰는 성능에 영향을 줄 수 있으므로, 실행 계획을 통해 성능을 확인하는 것이 좋습니다.
> - 과도한 뷰 사용은 오히려 데이터베이스 구조의 복잡성을 높일 수 있습니다.
