# 8. 다형성

## 개요
이 문서는 코틀린의 객체 지향 프로그래밍 핵심 개념 중 하나인 다형성(Polymorphism)에 대해 설명합니다. 다형성은 하나의 인터페이스나 부모 클래스를 통해 여러 다른 형태의 객체를 다룰 수 있게 하는 개념으로, 코드의 유연성과 재사용성을 크게 향상시킵니다.

## 다형성이란?
다형성(Polymorphism)은 그리스어로 "여러 가지 형태를 가질 수 있는 능력"을 의미합니다. 객체 지향 프로그래밍에서는 하나의 객체가 여러 타입을 가질 수 있거나, 하나의 메서드가 다양한 방식으로 동작할 수 있음을 의미합니다. 주로 상속과 인터페이스를 통해 구현됩니다.

## 다형성의 종류

### 1. 컴파일 타임 다형성 (정적 다형성) - 오버로딩 (Overloading)
메서드 오버로딩은 같은 이름의 메서드가 매개변수의 타입이나 개수가 다름에 따라 다르게 동작하는 것을 말합니다. 컴파일 시점에 어떤 메서드가 호출될지 결정됩니다.

```kotlin
class Calculator {
    fun add(a: Int, b: Int): Int {
        return a + b
    }

    fun add(a: Double, b: Double): Double {
        return a + b
    }

    fun add(a: Int, b: Int, c: Int): Int {
        return a + b + c
    }
}

fun main() {
    val calc = Calculator()
    println(calc.add(1, 2))       // Int 버전 호출
    println(calc.add(1.0, 2.0))   // Double 버전 호출
    println(calc.add(1, 2, 3))    // Int 3개 버전 호출
}
```

### 2. 런타임 다형성 (동적 다형성) - 오버라이딩 (Overriding)
메서드 오버라이딩은 부모 클래스에서 선언된 메서드를 자식 클래스에서 재정의하여 사용하는 것을 말합니다. 런타임에 실제 객체의 타입에 따라 호출될 메서드가 결정됩니다.

```kotlin
open class Animal {
    open fun sound() {
        println("동물이 소리를 냅니다.")
    }
}

class Dog : Animal() {
    override fun sound() {
        println("멍멍!")
    }
}

class Cat : Animal() {
    override fun sound() {
        println("야옹~")
    }
}

fun main() {
    val myDog: Animal = Dog() // Animal 타입으로 Dog 객체 참조
    val myCat: Animal = Cat() // Animal 타입으로 Cat 객체 참조

    myDog.sound() // 실제 객체인 Dog의 sound() 호출
    myCat.sound() // 실제 객체인 Cat의 sound() 호출

    val animals = listOf(Dog(), Cat(), Animal())
    for (animal in animals) {
        animal.sound() // 각 객체의 오버라이드된 sound() 호출
    }
}
```

## 다형성의 장점
*   **코드 재사용성**: 공통 인터페이스를 통해 다양한 객체를 다룰 수 있어 코드 재사용성이 높아집니다.
*   **유연성**: 새로운 클래스가 추가되어도 기존 코드를 수정할 필요 없이 확장 가능합니다 (개방-폐쇄 원칙).
*   **유지보수성**: 코드 변경이 필요한 경우, 특정 부분만 수정하면 되므로 유지보수가 용이합니다.
*   **가독성**: 추상화된 인터페이스를 통해 코드를 이해하기 쉬워집니다.
