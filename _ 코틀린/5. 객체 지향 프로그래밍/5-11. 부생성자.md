# 3. 부생성자

## 개요
이 문서는 코틀린의 부생성자(Secondary Constructor)에 대해 설명합니다. 부생성자는 클래스에 여러 가지 방법으로 객체를 초기화할 수 있도록 해주며, 특히 자바와의 상호 운용성이나 특정 초기화 로직이 필요할 때 유용하게 사용됩니다.

## 부생성자란?
코틀린 클래스는 주 생성자(Primary Constructor) 외에도 하나 이상의 부생성자를 가질 수 있습니다. 부생성자는 `constructor` 키워드를 사용하여 선언하며, 클래스 본문 내에 정의됩니다.

부생성자는 항상 주 생성자에 위임하거나, 주 생성자가 없는 경우 다른 부생성자에 위임해야 합니다. 이는 `this()` 키워드를 사용하여 수행됩니다.

## 사용 방법

### 기본 문법
```kotlin
class Person {
    var name: String
    var age: Int

    constructor(name: String, age: Int) {
        this.name = name
        this.age = age
        println("부생성자 호출: 이름 - $name, 나이 - $age")
    }

    constructor(name: String) : this(name, 0) { // 주 생성자가 없으므로 다른 부생성자에 위임
        println("부생성자 호출: 이름 - $name (나이 기본값 0)")
    }
}

fun main() {
    val person1 = Person("Alice", 30)
    val person2 = Person("Bob")
}
```

### 주 생성자와 함께 사용
클래스에 주 생성자가 있는 경우, 모든 부생성자는 직간접적으로 주 생성자에 위임해야 합니다.
```kotlin
class User(val id: String) { // 주 생성자
    var name: String
    var email: String

    constructor(id: String, name: String) : this(id) { // 주 생성자에 위임
        this.name = name
        this.email = ""
        println("부생성자 호출: ID - $id, 이름 - $name")
    }

    constructor(id: String, name: String, email: String) : this(id, name) { // 다른 부생성자에 위임
        this.email = email
        println("부생성자 호출: ID - $id, 이름 - $name, 이메일 - $email")
    }
}

fun main() {
    val user1 = User("user001")
    val user2 = User("user002", "Charlie")
    val user3 = User("user003", "David", "david@example.com")
}
```

## 주 생성자와의 관계
*   **주 생성자:** 클래스 헤더에 정의되며, 클래스의 주요 초기화 로직을 담당합니다. 프로퍼티 선언과 초기화를 간결하게 할 수 있습니다.
*   **부생성자:** `constructor` 키워드로 클래스 본문 내에 정의됩니다. 주 생성자가 있다면 반드시 주 생성자에 위임해야 합니다. 주 생성자가 없다면 다른 부생성자에 위임할 수 있습니다.
*   **초기화 블록 (`init`):** 주 생성자의 일부로 간주되며, 주 생성자가 호출될 때 실행됩니다. 부생성자가 주 생성자에 위임할 경우, `init` 블록이 먼저 실행된 후 부생성자의 본문이 실행됩니다.
