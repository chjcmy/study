# HashMap과 HashSet
#Kotlin/Collection

> [!INFO]
> `HashMap`과 `HashSet`은 코틀린 표준 라이브러리에서 제공하는, **해시(Hash)** 테이블을 기반으로 동작하는 컬렉션입니다. 이들은 삽입, 삭제, 검색 연산에서 평균적으로 **O(1)**의 매우 빠른 시간 복잡도를 가집니다.

## 1. HashMap

-   **정의**: **키(Key)-값(Value) 쌍**으로 데이터를 저장하는 가변(Mutable) 맵입니다.
-   **특징**:
    -   키는 고유해야 하며, 중복될 수 없습니다.
    -   값은 중복될 수 있습니다.
    -   삽입 순서를 보장하지 않습니다.
-   **생성**: `hashMapOf()` 함수를 사용합니다.

```kotlin
// <String, Int> 타입의 HashMap 생성
val scores = hashMapOf("Alice" to 90, "Bob" to 85)

// 추가 및 수정
scores["Charlie"] = 95 // 새로운 키-값 쌍 추가
scores["Alice"] = 92   // 기존 키의 값 수정

// 접근
println(scores["Alice"]) // 출력: 92

// 제거
scores.remove("Bob")

println(scores) // 출력: {Alice=92, Charlie=95}
```

## 2. HashSet

-   **정의**: **중복되지 않는** 고유한 원소들을 저장하는 가변(Mutable) 집합입니다.
-   **특징**:
    -   원소의 순서를 보장하지 않습니다.
    -   `null`을 하나만 저장할 수 있습니다.
-   **생성**: `hashSetOf()` 함수를 사용합니다.

```kotlin
// <String> 타입의 HashSet 생성
val tags = hashSetOf("Kotlin", "Programming")

// 추가
tags.add("Tutorial")
tags.add("Programming") // 중복된 원소이므로 무시됨

// 포함 여부 확인 (매우 빠름)
if ("Kotlin" in tags) {
    println("Kotlin tag exists.")
}

// 제거
tags.remove("Tutorial")

println(tags) // 출력: [Kotlin, Programming] (순서는 다를 수 있음)
```

## 3. 언제 무엇을 사용할까?

| 구분 | HashMap | HashSet |
| :--- | :--- | :--- |
| **데이터 구조** | 키-값 쌍 | 단일 값 |
| **주요 목적** | 키를 통해 값을 빠르게 **검색** | 원소의 **존재 여부**를 빠르게 확인하거나 **중복을 제거** |
| **예시** | 사용자 ID와 사용자 객체 매핑 | 이메일 주소 중복 체크 |

> [!TIP] `LinkedHashMap` / `LinkedHashSet`
> 만약 삽입된 순서를 유지해야 한다면, `linkedMapOf()` 또는 `linkedSetOf()`를 사용하여 `LinkedHashMap`이나 `LinkedHashSet`을 생성할 수 있습니다. 이들은 해시 테이블과 연결 리스트(Linked List)를 결합하여 순서를 기억하지만, `HashMap`/`HashSet`에 비해 약간의 성능 오버헤드가 있습니다.
