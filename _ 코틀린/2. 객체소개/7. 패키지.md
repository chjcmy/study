# 코틀린의 패키지 (Package)
#Kotlin/Concept

> [!INFO] 패키지란?
> **패키지(Package)**는 서로 관련 있는 클래스, 함수, 인터페이스 등을 하나의 **이름 공간(Namespace)** 안에 묶어 관리하는 메커니즘입니다. 패키지는 코드의 구조를 체계적으로 만들고, 이름 충돌을 방지하는 역할을 합니다.

## 1. 패키지 선언과 임포트

### 패키지 선언 (`package`)

-   코틀린 파일의 가장 위쪽에 `package` 키워드를 사용하여 해당 파일이 속할 패키지를 선언합니다.
-   패키지 이름은 일반적으로 소문자를 사용하며, 회사의 도메인을 역순으로 사용하는 것이 관례입니다. (예: `com.example.myapp`)
-   물리적인 디렉터리 구조와 패키지 구조를 일치시키는 것이 강력하게 권장됩니다.

```kotlin
// src/main/kotlin/com/example/utils/StringUtils.kt
package com.example.utils

fun toUpperCase(str: String): String {
    return str.uppercase()
}
```

### 임포트 (`import`)

-   다른 패키지에 있는 코드 요소를 사용하려면 `import` 키워드로 가져와야 합니다.

```kotlin
// src/main/kotlin/com/example/app/Main.kt
package com.example.app

// com.example.utils 패키지의 toUpperCase 함수를 임포트
import com.example.utils.toUpperCase

fun main() {
    val result = toUpperCase("hello") // 패키지 이름 없이 함수 사용 가능
    println(result) // 출력: HELLO
}
```

## 2. `import` 사용법

-   **개별 임포트**: 특정 클래스나 함수 하나만 가져옵니다.
    ```kotlin
    import kotlin.math.PI
    import kotlin.math.cos
    ```
-   **전체 임포트 (`*`)**: 해당 패키지의 모든 `public` 요소를 한 번에 가져옵니다.
    ```kotlin
    import kotlin.math.*
    ```
-   **별칭 사용 (`as`)**: 이름이 충돌하거나, 더 명확한 이름을 사용하고 싶을 때 `as` 키워드로 별칭을 부여할 수 있습니다.
    ```kotlin
    import java.util.Date as UtilDate
    import java.sql.Date as SqlDate

    val utilDate = UtilDate()
    val sqlDate = SqlDate(System.currentTimeMillis())
    ```

> [!TIP]
> `import` 없이도 패키지 전체 경로를 명시하여 코드를 사용할 수 있지만, 코드가 길어지고 가독성이 떨어지므로 `import`를 사용하는 것이 일반적입니다.
> ```kotlin
> println(kotlin.math.PI)
> ```

## 3. 기본 임포트 (Default Imports)

코틀린은 자주 사용되는 특정 패키지들을 모든 코틀린 파일에 자동으로 임포트해 줍니다. 따라서 이 패키지들에 포함된 요소들은 `import` 없이 바로 사용할 수 있습니다.

-   `kotlin.*`
-   `kotlin.annotation.*`
-   `kotlin.collections.*`
-   `kotlin.io.*`
-   `kotlin.text.*`
-   ... 등
