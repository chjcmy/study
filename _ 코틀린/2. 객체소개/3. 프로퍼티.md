# 코틀린의 프로퍼티 (Property)
#Kotlin/Property #Kotlin/ObjectOriented

> [!INFO] 프로퍼티란?
> **프로퍼티(Property)**는 클래스, 인터페이스, 또는 객체 내부에 선언되어 **객체의 상태(데이터)**를 저장하거나 나타내는 `val` 또는 `var` 변수입니다.

## 1. 프로퍼티의 특징

-   **`val` vs `var`**:
    -   `val` (value): **읽기 전용** 프로퍼티. 한 번 초기화되면 값을 변경할 수 없습니다. (Getter만 존재)
    -   `var` (variable): **변경 가능** 프로퍼티. 값을 다시 할당할 수 있습니다. (Getter와 Setter 모두 존재)
-   **독립적인 저장 공간**: 클래스로부터 생성된 각 객체(인스턴스)는 자신만의 프로퍼티 값을 저장할 독립적인 메모리 공간을 가집니다.
-   **접근**: 객체 변수 뒤에 점(`.`)을 찍어 프로퍼티에 접근합니다. (`myObject.myProperty`)

```kotlin
class Cup {
    var percentFull = 0 // Int 타입 프로퍼티 선언 및 초기화
}

fun main() {
    val c1 = Cup()
    c1.percentFull = 50 // c1 객체의 상태 변경

    val c2 = Cup()
    c2.percentFull = 70 // c2 객체의 상태 변경

    println("Cup 1: ${c1.percentFull}%") // 출력: Cup 1: 50%
    println("Cup 2: ${c2.percentFull}%") // 출력: Cup 2: 70%
}
```

## 2. 멤버 함수와 프로퍼티

클래스 내부의 멤버 함수는 점 표기법 없이 클래스 내의 다른 프로퍼티나 함수에 직접 접근할 수 있습니다.

```kotlin
class Cup {
    var percentFull = 0
    val max = 100

    fun add(increase: Int): Int {
        percentFull += increase
        if (percentFull > max) { // 클래스 내부 프로퍼티에 직접 접근
            percentFull = max
        }
        return percentFull
    }
}
```

## 3. 참조와 객체 상태

> [!IMPORTANT] `val`은 참조 불변, 객체는 가변
> `val`로 선언된 변수는 다른 객체를 가리키도록 **참조를 변경할 수 없다**는 의미이지, 그 참조가 가리키는 **객체의 내부 상태(프로퍼티)가 불변**이라는 의미는 아닙니다.

```kotlin
class House {
    var sofa: String = ""
}

fun main() {
    val myHouse = House() // myHouse는 House 객체를 참조
    myHouse.sofa = "Simple Sofa" // 객체의 내부 상태(sofa 프로퍼티)는 변경 가능

    // myHouse = House() // 컴파일 오류! val 변수는 다른 객체를 참조할 수 없음
}
```

### 참조 공유

두 변수가 동일한 객체를 참조하는 경우, 한 변수를 통해 객체의 상태를 변경하면 다른 변수에도 그 변경 사항이 반영됩니다.

```kotlin
val kitchen1 = Kitchen()
val kitchen2 = kitchen1 // kitchen2는 kitchen1과 동일한 객체를 참조

kitchen1.table = "Square Table"

println(kitchen2.table) // 출력: Square Table
```
