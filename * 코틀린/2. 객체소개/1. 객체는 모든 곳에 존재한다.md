# 코틀린과 객체
#Kotlin/ObjectOriented #Kotlin/Concept

> [!INFO] 코틀린의 모든 것은 객체다
> 코틀린은 완전한 객체 지향 언어입니다. 자바와 달리 `int`, `double`과 같은 원시 타입(Primitive Type)이 존재하지 않으며, `Int`, `Double` 등 모든 데이터 타입이 객체(Object)입니다.
>
> 이는 모든 변수에 대해 멤버 함수나 프로퍼티를 호출할 수 있음을 의미합니다.

## 1. 객체(Object)란?

-   **정의**: 객체는 **데이터(프로퍼티)**와 그 데이터를 처리하는 **동작(멤버 함수)**을 하나로 묶은 소프트웨어의 기본 단위입니다.
-   **인스턴스**: 객체는 특정 클래스의 **인스턴스(instance)**, 즉 '실체'입니다. 클래스가 '설계도'라면, 객체는 그 설계도로 만들어진 '제품'에 비유할 수 있습니다.

```kotlin
// IntRange 클래스의 인스턴스(객체)를 두 개 생성
val r1 = IntRange(0, 10)
val r2 = IntRange(5, 7)

// 각 객체는 고유한 데이터를 가짐
println(r1) // 출력: 0..10
println(r2) // 출력: 5..7
```

## 2. 멤버 함수와 프로퍼티

-   **멤버 함수(Member Function)**: 특정 클래스에 속한 객체만 호출할 수 있는 함수입니다. 객체의 데이터를 조작하거나 특정 연산을 수행합니다.
-   **프로퍼티(Property)**: 객체가 가지는 데이터입니다.

```kotlin
val r = IntRange(0, 10)

// IntRange 객체의 멤버 함수 sum() 호출
println(r.sum()) // 출력: 55

val s = "AbcD"

// String 객체의 멤버 함수 reversed()와 프로퍼티 length 호출
println(s.reversed()) // 출력: DcbA
println(s.length)     // 출력: 4
```

> [!TIP] IDE 활용하기
> IntelliJ IDEA와 같은 IDE에서 객체 변수 뒤에 점(`.`)을 찍으면, 해당 객체에서 사용할 수 있는 모든 멤버 함수와 프로퍼티 목록을 확인할 수 있습니다. 이를 '코드 완성(Code Completion)' 기능이라고 합니다.

## 3. 모든 것이 객체이기 때문에 가능한 것

코틀린에서는 원시 타입처럼 보이는 숫자나 문자열조차 객체이므로, 관련된 유용한 함수들을 바로 사용할 수 있습니다.

### 타입 변환

`toInt()`, `toDouble()`, `toString()` 등 모든 타입 변환은 객체의 멤버 함수를 호출하는 방식으로 이루어집니다.

```kotlin
val s = "123"
val i = s.toInt() // String 객체의 toInt() 함수 호출

val num = 1
val den = 2
// Int 객체의 toDouble() 함수를 호출하여 정확한 나눗셈 결과 얻기
val fraction = num.toDouble() / den.toDouble()
println(fraction) // 출력: 0.5
```
