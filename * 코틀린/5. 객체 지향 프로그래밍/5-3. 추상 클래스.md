# 6. 추상 클래스

## 개요
이 문서는 코틀린의 추상 클래스(Abstract Class)에 대해 설명합니다. 추상 클래스는 객체 지향 프로그래밍에서 공통된 특성과 동작을 정의하면서도, 일부 구현을 자식 클래스에게 위임하여 유연성을 제공하는 중요한 개념입니다.

## 추상 클래스란?
추상 클래스는 `abstract` 키워드를 사용하여 선언되는 클래스입니다. 이 클래스는 하나 이상의 추상 멤버(추상 메서드 또는 추상 프로퍼티)를 포함할 수 있으며, 추상 멤버는 선언만 있고 구현은 없는 형태입니다. 추상 클래스는 직접 인스턴스화할 수 없으며, 반드시 자식 클래스에서 상속받아 추상 멤버들을 구현해야만 인스턴스화할 수 있습니다.

### 특징
*   `abstract` 키워드로 선언됩니다.
*   추상 메서드와 추상 프로퍼티를 가질 수 있습니다.
*   일반(비추상) 메서드와 프로퍼티도 가질 수 있습니다.
*   직접 객체를 생성할 수 없습니다 (인스턴스화 불가).
*   다른 클래스에 의해 상속되어야 합니다.
*   자식 클래스는 부모 추상 클래스의 모든 추상 멤버를 `override` 키워드를 사용하여 반드시 구현해야 합니다.

## 사용 방법 및 예제

```kotlin
// 추상 클래스 선언
abstract class Shape {
    abstract val name: String // 추상 프로퍼티

    abstract fun calculateArea(): Double // 추상 메서드
    abstract fun calculatePerimeter(): Double // 추상 메서드

    fun printInfo() { // 일반 메서드
        println("도형 이름: $name")
    }
}

// 추상 클래스를 상속받는 자식 클래스 (원)
class Circle(val radius: Double) : Shape() {
    override val name: String = "원"

    override fun calculateArea(): Double {
        return Math.PI * radius * radius
    }

    override fun calculatePerimeter(): Double {
        return 2 * Math.PI * radius
    }
}

// 추상 클래스를 상속받는 자식 클래스 (사각형)
class Rectangle(val width: Double, val height: Double) : Shape() {
    override val name: String = "사각형"

    override fun calculateArea(): Double {
        return width * height
    }

    override fun calculatePerimeter(): Double {
        return 2 * (width + height)
    }
}

fun main() {
    val circle: Shape = Circle(5.0)
    val rectangle: Shape = Rectangle(4.0, 6.0)

    circle.printInfo()
    println("원의 면적: ${circle.calculateArea()}")
    println("원의 둘레: ${circle.calculatePerimeter()}")

    rectangle.printInfo()
    println("사각형의 면적: ${rectangle.calculateArea()}")
    println("사각형의 둘레: ${rectangle.calculatePerimeter()}")

    // val shape = Shape() // 컴파일 오류: 추상 클래스는 인스턴스화할 수 없습니다.
}
```

## 추상 클래스와 인터페이스의 차이점
| 특징 | 추상 클래스 (Abstract Class) | 인터페이스 (Interface) |
| :--- |
| **구현** | 추상 멤버와 일반 멤버 모두 가질 수 있음 | 추상 멤버만 가질 수 있었으나, 코틀린 1.0부터 기본 구현을 포함할 수 있게 됨 |
| **상태 (프로퍼티)** | 상태를 저장하는 프로퍼티(필드)를 가질 수 있음 | 상태를 저장하는 프로퍼티(필드)를 가질 수 없음 (추상 프로퍼티는 가능) |
| **생성자** | 생성자를 가질 수 있음 | 생성자를 가질 수 없음 |
| **상속** | 단일 상속만 가능 | 여러 인터페이스 구현 가능 |
| **접근 제어자** | `public`, `protected`, `private`, `internal` 사용 가능 | `public`만 가능 (기본) |
| **주요 용도** | `is-a` 관계에서 공통된 구현과 미완성된 부분을 함께 제공할 때 | `can-do` 관계에서 특정 동작을 강제하고 다중 상속의 이점을 얻을 때 |
