# 16. 객체

## 개요
이 문서는 코틀린의 객체 선언(Object Declaration)에 대해 설명합니다. 코틀린의 `object` 키워드를 사용하면 싱글톤 패턴을 간결하게 구현하거나, 익명 객체를 생성하는 등 다양한 상황에서 유용하게 활용할 수 있습니다.

## 객체 선언 (Object Declaration)이란?
코틀린에서 `object` 키워드를 사용하여 클래스를 정의함과 동시에 해당 클래스의 단일 인스턴스를 생성하는 기능입니다. 이는 자바의 `static` 멤버나 싱글톤 패턴을 코틀린스럽게 구현하는 방법 중 하나입니다.

### 특징
*   **싱글톤**: `object`로 선언된 클래스는 애플리케이션 전체에서 단 하나의 인스턴스만 존재합니다.
*   **지연 초기화**: `object`는 처음 접근될 때 초기화됩니다 (lazy initialization).
*   **상속 및 인터페이스 구현**: `object`는 다른 클래스를 상속받거나 인터페이스를 구현할 수 있습니다.
*   **생성자 없음**: `object`는 생성자를 가질 수 없습니다. 초기화 로직은 `init` 블록에 작성합니다.

## 사용 방법

### 1. 싱글톤 객체
가장 일반적인 사용 사례는 싱글톤 패턴을 구현하는 것입니다.
```kotlin
object Logger {
    init {
        println("Logger 객체 초기화됨")
    }
    fun log(message: String) {
        println("[LOG] $message")
    }
}

fun main() {
    Logger.log("애플리케이션 시작") // Logger 객체가 이때 초기화됨
    Logger.log("데이터 처리 중")
    // Logger 객체는 단 하나만 존재하며, 매번 동일한 인스턴스에 접근
}
```

### 2. 익명 객체 (Anonymous Object)
특정 인터페이스나 클래스를 일시적으로 구현하는 익명 객체를 생성할 때도 `object` 키워드를 사용합니다.
```kotlin
interface EventListener {
    fun onEvent(event: String)
}

fun registerListener(listener: EventListener) {
    listener.onEvent("등록 이벤트 발생!")
}

fun main() {
    registerListener(object : EventListener {
        override fun onEvent(event: String) {
            println("익명 객체에서 이벤트 처리: $event")
        }
    })
}
```

### 3. 팩토리 메서드 (Factory Method)
클래스 내부에 `object`를 선언하여 팩토리 메서드를 제공할 수 있습니다.
```kotlin
class MyFactory private constructor(val value: String) {
    companion object { // 동반 객체 내부에 object 선언
        fun create(type: String): MyFactory {
            return when (type) {
                "A" -> MyFactory("Type A")
                "B" -> MyFactory("Type B")
                else -> MyFactory("Default Type")
            }
        }
    }
}

fun main() {
    val instanceA = MyFactory.create("A")
    println(instanceA.value)
}
```

## 동반 객체(Companion Object)와의 차이점
| 특징 | 객체 선언 (Object Declaration) | 동반 객체 (Companion Object) |
| :--- | :--- | :--- |
| **선언 위치** | 독립적으로 선언되거나 클래스 내부에 중첩 | 클래스 내부에 `companion` 키워드와 함께 선언 |
| **인스턴스 수** | 항상 단 하나의 인스턴스 (싱글톤) | 클래스당 단 하나의 인스턴스 (싱글톤) |
| **접근 방식** | `ObjectName.member` | `ClassName.member` (클래스 이름으로 직접 접근) |
| **주요 용도** | 유틸리티 객체, 싱글톤, 익명 객체 | 팩토리 메서드, 클래스 관련 유틸리티 함수/상수 |
