# 18. 동반 객체

## 개요
이 문서는 코틀린의 동반 객체(Companion Object)에 대해 설명합니다. 동반 객체는 클래스 내부에 정의되는 특별한 객체로, 해당 클래스의 인스턴스 없이도 접근할 수 있는 멤버(함수, 프로퍼티)를 가질 수 있게 해줍니다. 이는 자바의 `static` 멤버와 유사한 기능을 코틀린에서 제공하는 방법입니다.

## 동반 객체란?
코틀린에서 클래스 내부에 `companion object` 키워드를 사용하여 선언하는 객체입니다. 동반 객체의 멤버는 해당 클래스의 이름으로 직접 접근할 수 있습니다. 마치 클래스의 정적(static) 멤버처럼 보이지만, 실제로는 클래스 내부에 정의된 단일 인스턴스 객체입니다.

### 특징
*   클래스당 하나의 동반 객체만 가질 수 있습니다.
*   클래스 이름으로 직접 접근할 수 있습니다 (예: `MyClass.myFunction()`).
*   인터페이스를 구현하거나 상위 클래스를 상속받을 수 있습니다.
*   확장 함수를 가질 수 있습니다.

## 사용 방법

### 기본 문법
```kotlin
class MyClass {
    companion object {
        const val PI = 3.141592 // 상수
        fun create(): MyClass { // 팩토리 메서드
            println("MyClass 인스턴스 생성")
            return MyClass()
        }
    }

    fun doSomething() {
        println("인스턴스 메서드 호출")
    }
}

fun main() {
    println("PI 값: ${MyClass.PI}") // 클래스 이름으로 동반 객체 멤버 접근
    val instance = MyClass.create()
    instance.doSomething()
}
```

### 인터페이스 구현
동반 객체는 인터페이스를 구현할 수 있습니다.
```kotlin
interface Factory<T> {
    fun create(): T
}

class MyData {
    companion object : Factory<MyData> {
        override fun create(): MyData {
            println("MyData 인스턴스 생성 (Factory 패턴)")
            return MyData()
        }
    }
}

fun main() {
    val data = MyData.create()
}
```

## 정적 멤버와의 비교 (Java `static` vs Kotlin `companion object`)
| 구분 | Java `static` | Kotlin `companion object` |
| :--- | :--- | :--- |
| **구현 방식** | 클래스 레벨의 정적 멤버 | 클래스 내부에 정의된 단일 객체 인스턴스 |
| **접근 방식** | `ClassName.staticMember` | `ClassName.companionObjectMember` (또는 `ClassName.Member`로 생략 가능) |
| **유연성** | 인터페이스 구현 불가, 상속 불가 | 인터페이스 구현 가능, 상위 클래스 상속 가능 |
| **확장성** | 확장 함수 불가 | 확장 함수 가능 |
| **용도** | 유틸리티 함수, 상수 | 팩토리 메서드, 유틸리티 함수, 인터페이스 구현, 확장 함수 대상 |
