# 함수 오버로딩 (Overloading)
#Kotlin/Function

> [!INFO] 오버로딩이란?
> **오버로딩(Overloading)**은 **같은 이름**의 함수를 여러 개 정의하되, **매개변수(파라미터)의 개수나 타입**을 다르게 하여 서로 다른 함수로 인식하게 하는 기능입니다.

이를 통해 함수 사용자는 동일한 함수 이름을 사용하면서, 다양한 종류의 인자를 전달하여 원하는 동작을 수행할 수 있습니다.

## 1. 오버로딩의 조건

컴파일러는 다음 조건들을 기준으로 어떤 함수를 호출할지 결정합니다.
-   **매개변수의 개수**가 달라야 합니다.
-   매개변수의 개수가 같더라도, **타입**이 달라야 합니다.
-   매개변수의 개수와 타입이 같더라도, **순서**가 달라야 합니다.

> [!IMPORTANT]
> 함수의 **반환 타입**만 다른 경우는 오버로딩으로 인정되지 않으며, 컴파일 오류가 발생합니다.

## 2. 오버로딩 예시

### 매개변수 개수가 다른 경우
```kotlin
fun printMessage() {
    println("Hello, Guest!")
}

fun printMessage(name: String) {
    println("Hello, $name!")
}

fun main() {
    printMessage()      // 출력: Hello, Guest!
    printMessage("Alice") // 출력: Hello, Alice!
}
```

### 매개변수 타입이 다른 경우
```kotlin
fun add(a: Int, b: Int): Int {
    println("Adding two integers")
    return a + b
}

fun add(a: Double, b: Double): Double {
    println("Adding two doubles")
    return a + b
}

fun main() {
    println(add(3, 5))       // 출력: Adding two integers -> 8
    println(add(2.5, 4.5))   // 출력: Adding two doubles -> 7.0
}
```

## 3. 오버로딩 vs 디폴트 인자

> [!TIP]
> 코틀린에서는 **디폴트 인자(Default Arguments)**를 사용하여 불필요한 오버로딩을 상당수 줄일 수 있습니다.

-   **오버로딩을 사용한 경우**
    ```kotlin
    fun greet() { ... }
    fun greet(name: String) { ... }
    fun greet(name: String, message: String) { ... }
    ```
-   **디폴트 인자를 사용한 경우**
    ```kotlin
    fun greet(name: String = "Guest", message: String = "Hello") {
        println("$message, $name!")
    }

    fun main() {
        greet()                  // 출력: Hello, Guest!
        greet("Alice")           // 출력: Hello, Alice!
        greet("Bob", "Hi")       // 출력: Hi, Bob!
        greet(message = "Welcome") // 출력: Welcome, Guest!
    }
    ```
    하나의 함수로 여러 오버로딩 함수의 역할을 대체할 수 있어 코드가 훨씬 간결해집니다.
