# 식(Expression)과 문(Statement)
#Kotlin/Basic #Kotlin/Concept

> [!INFO]
> 프로그래밍 언어의 구성 요소를 이해하는 데 있어 '식(Expression)'과 '문(Statement)'의 차이를 아는 것은 매우 중요합니다. 코틀린은 **식이 중심이 되는(Expression-oriented)** 언어입니다.

## 1. 문 (Statement)

-   **정의**: 어떤 동작을 수행하도록 지시하는 코드 단위이지만, **결과 값을 만들어내지 않습니다.**
-   **목적**: 주로 프로그램의 상태를 변경하는 **부수 효과(Side Effect)**를 일으키기 위해 사용됩니다.
-   **예시**:
    -   변수 선언 (`val a = 10`)
    -   `println("Hello")` 호출
    -   자바의 `if`, `for` 등 대부분의 제어문

```java
// 자바의 예시
int result;
if (a > b) {
    result = a; // if문은 값을 반환하지 않으므로, 외부 변수에 값을 할당해야 함
} else {
    result = b;
}
```

## 2. 식 (Expression)

-   **정의**: 평가(evaluate)된 후 **항상 결과 값을 만들어내는** 코드 단위입니다.
-   **목적**: 값을 계산하고 생성하는 것입니다. 식의 결과는 다른 변수에 할당되거나, 다른 식의 일부가 될 수 있습니다.
-   **예시**:
    -   `1 + 2`
    -   `"Hello"`
    -   `myFunction()` 호출

## 3. 코틀린의 접근 방식: 모든 것은 식이다

> [!IMPORTANT] 코틀린의 특징
> 코틀린에서는 `if`, `when`, `try-catch` 등 전통적으로 '문'으로 취급되던 대부분의 제어 구조가 '식'으로 취급됩니다.

이러한 특징 덕분에 코드는 더 간결해지고, 불필요한 임시 변수 선언을 줄일 수 있습니다.

```kotlin
// 코틀린의 예시
val result = if (a > b) {
    println("a is larger")
    a // if 식의 결과 값
} else {
    println("b is larger or equal")
    b // else 식의 결과 값
}
```

### `Unit` 타입

코틀린에서는 값을 반환하지 않는 것처럼 보이는 함수도 실제로는 `Unit`이라는 특별한 객체를 반환합니다. 이는 "반환할 의미 있는 값이 없음"을 나타내는 싱글턴(Singleton) 객체입니다. 자바의 `void`와 유사한 역할을 합니다.

```kotlin
fun sayHello() { // 반환 타입이 생략되면 ': Unit'으로 간주됨
    println("Hello")
}

val result = sayHello()
println(result) // 출력: kotlin.Unit
```

## 요약

| 구분 | 문 (Statement) | 식 (Expression) |
| :--- | :--- | :--- |
| **결과 반환** | **하지 않음** | **항상 함** |
| **주요 역할** | 동작 수행, 상태 변경 | 값 계산, 결과 생성 |
| **코틀린에서**| 대입, `val`/`var` 선언 등 일부 | **대부분의 제어 구조 포함** |

> [!TIP] 좋은 코드 작성법
> - 부수 효과를 줄이기 위해 **식이 중심이 되는 프로그래밍**을 지향하는 것이 좋습니다.
> - 코틀린의 `if`, `when`을 식으로 적극 활용하면 코드가 더 간결하고 안전해집니다.
