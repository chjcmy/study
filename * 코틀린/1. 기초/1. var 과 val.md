# var와 val: 코틀린의 변수 선언
#Kotlin/Basic

> [!INFO]
> 코틀린에서는 변수를 선언할 때 `var`와 `val` 두 가지 키워드를 사용합니다. 이 둘의 가장 큰 차이는 **변경 가능성(Mutability)**에 있습니다.

## 1. `var` (variable)

-   **변경 가능한 변수**를 선언합니다.
-   선언 후에도 값을 여러 번 다시 할당할 수 있습니다.
-   주로 변경이 필요한 상태 값을 관리할 때 사용됩니다.

```kotlin
var name = "Kotlin"
println(name) // 출력: Kotlin

name = "Java" // 값 변경 가능
println(name) // 출력: Java
```

> [!CAUTION] `var` 사용 시 주의
> `var`는 값을 자유롭게 변경할 수 있어 편리하지만, 예상치 못한 상태 변경으로 인해 버그가 발생할 수 있습니다. 따라서 꼭 필요한 경우가 아니라면 `val` 사용을 우선하는 것이 좋습니다.

## 2. `val` (value)

-   **변경 불가능한(immutable) 읽기 전용 변수**를 선언합니다.
-   한 번 초기화하면 다른 값으로 재할당할 수 없습니다. (자바의 `final` 키워드와 유사)
-   **불변성(Immutability)**을 보장하여 데이터의 안정성을 높여줍니다.

```kotlin
val language = "Kotlin"
println(language) // 출력: Kotlin

// language = "Java" // 컴파일 오류: Val cannot be reassigned
```

> [!TIP] `val`과 불변성
> `val`은 변수에 할당된 **참조**가 불변임을 보장하는 것이지, 그 참조가 가리키는 **객체의 내부 상태**까지 불변으로 만드는 것은 아닙니다.
> ```kotlin
> val list = mutableListOf("a", "b", "c")
> // list = mutableListOf("d", "e") // 에러: 참조 변경 불가
>
> list.add("d") // 참조가 가리키는 객체의 내부 상태는 변경 가능
> println(list) // 출력: [a, b, c, d]
> ```

## var vs val: 언제 무엇을 써야 할까?

| 특징 | `var` (변경 가능) | `val` (변경 불가능) |
| :--- | :--- | :--- |
| **값 재할당** | **가능** | **불가능** |
| **주 사용처** | 루프 카운터, 변경이 잦은 상태 값 | 클래스 프로퍼티, 한 번 정해지면 변하지 않는 값 |
| **안정성** | 낮음 | 높음 |

> [!IMPORTANT] 코틀린의 철학: 불변성을 우선하라
> 코틀린은 함수형 프로그래밍의 영향을 받아 **불변성을 지향**합니다. 부수 효과(Side Effect)를 줄이고 예측 가능한 코드를 작성하기 위해, **기본적으로 모든 변수를 `val`로 선언**하고, 꼭 필요한 경우에만 `var`로 변경하는 습관을 들이는 것이 좋습니다.
