# if 표현식 (if Expression)
#Kotlin/Basic #Kotlin/ControlFlow

> [!INFO] 식(Expression) vs 문(Statement)
> - **문 (Statement)**: 어떤 동작을 수행하지만, 값을 반환하지 않습니다. (예: 자바의 `if`문, `for`문)
> - **식 (Expression)**: 어떤 동작을 수행하고, **결과로 값을 반환**합니다.
>
> 코틀린에서 `if`는 문이 아니라 **식**입니다. 따라서 `if` 자체가 값을 가질 수 있으며, 자바의 삼항 연산자(`condition ? a : b`)를 완벽하게 대체합니다.

## 1. 식으로 사용하기

`if`가 값을 반환하므로, 그 결과를 변수에 직접 할당할 수 있습니다.

```kotlin
val a = 10
val b = 20

// if 식의 결과를 max 변수에 할당
val max = if (a > b) a else b

println(max) // 출력: 20
```

`if`의 각 브랜치(`{...}`)는 여러 줄로 구성될 수 있으며, 이 경우 **블록의 마지막 표현식**이 그 브랜치의 결과 값이 됩니다.

```kotlin
val max = if (a > b) {
    println("a is greater")
    a // 이 값이 브랜치의 결과
} else {
    println("b is greater or equal")
    b // 이 값이 브랜치의 결과
}
```

## 2. `else if` 사용

여러 조건을 연달아 검사할 수 있습니다.

```kotlin
val score = 85
val grade = if (score >= 90) {
    "A"
} else if (score >= 80) {
    "B"
} else if (score >= 70) {
    "C"
} else {
    "F"
}
println(grade) // 출력: B
```

> [!TIP]
> 조건이 많아지면 `if-else if` 체인보다는 `when` 표현식을 사용하는 것이 가독성에 더 좋습니다.

## 3. 문(Statement)으로 사용하기

물론, 전통적인 방식처럼 값을 반환하지 않는 일반적인 조건문으로도 사용할 수 있습니다.

```kotlin
val number = -10
if (number > 0) {
    println("Positive number")
} else {
    println("Non-positive number") // 이 코드가 실행됨
}
```

> [!IMPORTANT] `if`를 식으로 사용할 때
> `if`를 식으로 사용하여 변수에 값을 할당하는 경우, **`else`는 반드시 있어야 합니다.** 모든 가능한 경우에 대해 값이 반환되어야 하기 때문입니다.
> ```kotlin
> val result = if (condition) "Yes" // 컴파일 오류! else가 없음
> ```
