# 배열과 리스트 (Array & List)

## 1. 배열 (Array)

### 1.1 특징
- **연속된 메모리 공간**에 데이터가 저장됩니다.
- 인덱스(Index)를 사용하여 **O(1)** 시간 복잡도로 데이터에 직접 접근(Random Access)할 수 있습니다.
- 크기가 고정되어 있어(Static Array), 생성 후에는 크기를 변경할 수 없습니다. (C/Java 기본 배열 기준)
- 데이터 중간에 삽입/삭제 시, 데이터를 밀거나 당겨야 하므로 **O(n)** 시간이 걸립니다.

### 1.2 파이썬에서의 배열
- 파이썬의 `list`는 실제로는 **동적 배열(Dynamic Array)** 로 구현되어 있습니다.
- 크기가 가변적이며, 배열의 장점(인덱스 접근)과 리스트의 편리함을 모두 가집니다.

```python
# 파이썬 리스트 사용 (배열처럼 사용)
arr = [1, 2, 3, 4]
print(arr[2])  # 인덱스 접근 O(1)
arr.append(5)  # 맨 뒤 추가 O(1) ~ Amortized
```

## 2. 연결 리스트 (Linked List)

### 2.1 특징
- 메모리에 데이터가 불연속적으로 저장되며, **포인터(Pointer)** 를 통해 다음 데이터를 연결합니다.
- 물리적 순서와 논리적 순서가 일치하지 않을 수 있습니다.
- 인덱스가 없으므로 데이터 접근 시 처음(Head)부터 순회해야 하여 **O(n)** 시간이 걸립니다.
- 데이터 삽입/삭제 시, 포인터만 변경하면 되므로 위치만 안다면 **O(1)** 로 빠릅니다.

### 2.2 파이썬 구현 (Node 클래스)

```python
class Node:
    def __init__(self, data):
        self.data = data
        self.next = None

class LinkedList:
    def __init__(self):
        self.head = None
    
    def append(self, data):
        new_node = Node(data)
        if not self.head:
            self.head = new_node
            return
        current = self.head
        while current.next:
            current = current.next
        current.next = new_node
        
    def print_all(self):
        current = self.head
        while current:
            print(current.data, end=" -> ")
            current = current.next
        print("None")

# 사용 예시
ll = LinkedList()
ll.append(10)
ll.append(20)
ll.append(30)
ll.print_all() # 10 -> 20 -> 30 -> None
```

## 3. 비교 요약
| 특징 | 배열 (Array) | 연결 리스트 (Linked List) |
| :--- | :--- | :--- |
| **메모리 구조** | 연속적 | 불연속적 (포인터 연결) |
| **접근 속도** | 빠름 (O(1)) | 느림 (O(n)) |
| **삽입/삭제** | 느림 (O(n)) | 빠름 (O(1), 위치 아는 경우) |
| **크기** | 고정 (언어에 따라 다름) | 가변 |
| **용도** | 데이터 조회 빈번, 크기 고정 | 삽입/삭제 빈번, 크기 가변 |
